void NetTopologySuite.Tests.NUnit.Mathematics.DDIOTest.TestParse ( )

The Java double-precision constant 1.4 gives rise to a value which differs from the exact binary representation down around the 17th decimal place. Thus it will not compare exactly to the DoubleDouble representation of the same number. To avoid this, compute the expected value using full DD precision.
